# 두레이 업무 문서 관리 (생성 / 수정)

dooray-cli를 사용하여 팀 양식에 맞는 업무 문서를 두레이 프로젝트에 생성하거나 수정한다.

## 업무 문서 양식

```
제목: [{프로젝트 코드}] {업무 핵심 제목}
담당자: AI서비스개발팀-정훈희
```

본문:
```markdown
# 개요
- {이 업무를 왜 하게 되었는지 이유, 배경을 간단하게 요약하여 설명한다.}

# 내용
- {이 업무에 대한 내용을 자세하게 구조화하여 설명한다.}

# 참고
- {이 업무에 관련된 참고할 정보나 자료 또는 URL링크등의 정보를 추가 기입한다.}
```

---

## 모드 판별

사용자의 요청에서 모드를 판별한다.

| 키워드 | 모드 |
|--------|------|
| 등록, 생성, 작성, create | **생성 모드** |
| 수정, 변경, 업데이트, edit, update | **수정 모드** |

---

## 생성 모드

### 1단계: 프로젝트 확인

사용자가 프로젝트를 지정한 경우에도 반드시 dooray-cli로 실제 존재하는지 확인한다.

```bash
dooray project list --search "<프로젝트 코드 또는 키워드>" --json
```

- 검색 결과가 여러 개면 목록을 보여주고 선택하게 한다
- 검색 결과가 없으면 키워드를 바꿔 재검색하거나 사용자에게 확인한다
- 프로젝트 코드가 확정되면 사용자에게 확인받고 다음 단계로 진행한다

### 2단계: 정보 수집

**부족한 정보가 있으면 반드시 먼저 질문한다.**

필수 정보:
- **업무 내용**: 무엇을 해야 하는지, 왜 해야 하는지 (개요 + 내용 작성에 필요)

선택 정보:
- **참고 자료**: 관련 URL, 문서, 자료 등
- **마감일**: 마감 기한이 있는 경우
- **우선순위**: highest, high, normal(기본), low, lowest

### 3단계: 문서 작성 및 확인

수집한 정보를 바탕으로 업무 문서를 양식에 맞게 작성한다.

1. 제목은 `[{프로젝트 코드}] {핵심 요약}` 형식
2. 본문은 개요/내용/참고 3개 섹션으로 구조화
3. **등록 전에 작성된 문서를 보여주고 사용자 확인을 받는다**

### 4단계: 두레이 등록

```bash
cat <<'EOF' > /tmp/dooray-task-body.md
# 개요
- ...

# 내용
- ...

# 참고
- ...
EOF

dooray post create <project> \
  --subject "[{프로젝트코드}] {제목}" \
  --body-file /tmp/dooray-task-body.md \
  --to "정훈희" \
  --json

rm -f /tmp/dooray-task-body.md
```

등록 성공 시:
- `post create`는 ID만 반환하므로, `post list`로 업무 번호(number)를 확인한다 (`post search`는 인덱싱 지연으로 생성 직후 조회가 안 될 수 있음)
  ```bash
  dooray post list <project> --json
  # → 최신 업무가 맨 위: "number": 142, "taskNumber": "@hhjung/142"
  ```
- 업무 번호를 사용자에게 알려준다
- 브랜치가 필요한 경우 `feature/{프로젝트코드}/{업무번호}` 형식을 안내한다

## 수정 모드

### 1단계: 대상 업무 확인

업무 번호를 알고 있으면 바로 조회, 모르면 검색하여 특정한다.

```bash
# 업무 번호를 아는 경우
dooray post get <project> <number> --json

# 모르는 경우 검색
dooray post search <project> "<keyword>" --json
```

- 검색 결과가 여러 개면 목록을 보여주고 선택하게 한다
- 기존 본문을 조회하여 현재 내용을 파악한다

### 2단계: 변경 내용 확인

사용자에게 어떤 부분을 수정할지 확인한다.

- 제목만 변경
- 본문 특정 섹션(개요/내용/참고) 변경
- 전체 본문 재작성

**수정 시에도 팀 양식(개요/내용/참고 구조)을 유지한다.**

### 3단계: 수정안 확인

기존 내용과 변경될 내용을 비교하여 보여주고 **사용자 확인을 받는다**.

### 4단계: 두레이 수정

```bash
# 제목만 수정
dooray post edit <project> <number> --subject "[{프로젝트코드}] {새 제목}" --json

# 본문 수정
cat <<'EOF' > /tmp/dooray-task-body.md
# 개요
- ...

# 내용
- ...

# 참고
- ...
EOF

dooray post edit <project> <number> --body-file /tmp/dooray-task-body.md --json

rm -f /tmp/dooray-task-body.md
```

## 주의사항

- 정보가 부족하면 **추측하지 말고 반드시 사용자에게 질문**한다
- 프로젝트는 반드시 dooray-cli로 실제 존재 여부를 확인한다
- 생성/수정 전 반드시 사용자 확인을 거친다
- `--json` 플래그를 사용하여 결과를 파싱한다
- 수정 시 기존 양식(개요/내용/참고)을 깨뜨리지 않는다
- **프로젝트 접근 실패 시 임의로 다른 프로젝트를 선택하지 말고, 반드시 사용자에게 확인을 받는다**
- **개인 프로젝트(`@` 접두사)는 캐시에 없을 수 있다. 접근 실패 시 `dooray cache clear` 후 `dooray project list --type private`로 캐시를 갱신한다.**
- **본문은 반드시 `--body-file`을 사용한다. `--body`에 `\n`을 넣으면 리터럴 문자열로 들어간다.**
- **업무 조회/수정 시 ID가 아닌 업무 번호(number)를 사용한다. `post create` 반환값에는 number가 없으므로 `post list`로 확인한다 (`post search`는 인덱싱 지연 있음).**
